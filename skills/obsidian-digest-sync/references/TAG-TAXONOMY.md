# Tag Taxonomy — obsidian-digest-sync

Seven namespaces. All hierarchical (Obsidian nested tag syntax).
**Before creating any `#concept/` or `#data/` tag, call `mcp__obsidian-files__list_all_tags`
and prefer an existing tag over a new one to prevent fragmentation.**

> **Customize this file for your field.** The `#source/`, `#method/`, and `#status/`
> namespaces are field-agnostic and can be used as-is. The `#area/`, `#concept/`,
> `#data/`, and `#project/` seed lists below are **examples** — replace them with your
> own research areas, theoretical concepts, datasets, and active projects. The skill
> grows these lists over time as new tags appear in your digests.

## `#source/` — Origin of the note (field-agnostic, keep as-is)
- `#source/digest` — created by obsidian-digest-sync from a journal digest
- `#source/zotero` — created from ZotPilot ingest
- `#source/session` — created during a Claude research session
- `#source/manual` — created by the user directly
